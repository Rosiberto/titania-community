# API Data Submission

## Submission Endpoint

All telemetry data generated by devices must be sent to the following endpoint:

```text
http://<platform-host>/api/send
````

The TiTaniA Platform automatically routes incoming telemetry data to the appropriate IoT Agent according to the communication protocol configured for the device.

---

## Request Format

The request body must be a JSON document containing the following fields:

| Field         | Required | Description                                                                                                                                                 |
| ------------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `agent`       | No       | Communication mode used by the device. Supported values are `ul-http`, `ul-mqtt`, `json-http`, and `json-mqtt`. If omitted, the default value is `json-http`. |
| `apikey`      | Yes      | API Key associated with the Service Group.                                                                                                                  |
| `deviceid`    | Yes      | Unique identifier of the device.                                                                                                                            |
| `<attribute>` | Yes      | One or more telemetry attributes generated by sensors or internal device measurements.                                                                      |

All fields except `agent`, `apikey`, and `deviceid` are interpreted as telemetry attributes and forwarded to the selected IoT Agent.


## Example Request

```json
{
  "agent": "json-http",
  "apikey": "2b58f46a542446a49e2366f929918aa4",
  "deviceid": "sensor001",
  "temperature": 24.5,
  "humidity": 68,
  "battery": 95
}
```


## Processing Flow

### UltraLight over HTTP (`ul-http`)

The platform converts the JSON payload into the UltraLight format and forwards the data to the UltraLight IoT Agent using HTTP.

```text
Device → /api/send → UltraLight IoT Agent → Orion Context Broker
```

### UltraLight over MQTT (`ul-mqtt`)

The platform publishes telemetry data to the Mosquitto MQTT Broker using the UltraLight protocol.

```text
Device → /api/send → Mosquitto MQTT Broker → UltraLight IoT Agent → Orion Context Broker
```

### JSON over HTTP (`json-http`)

The platform forwards the telemetry payload to the JSON IoT Agent using HTTP.

```text
Device → /api/send → JSON IoT Agent → Orion Context Broker
```

### JSON over MQTT (`json-mqtt`)

The platform publishes telemetry data to the Mosquitto MQTT Broker using the JSON protocol.

```text
Device → /api/send → Mosquitto MQTT Broker → JSON IoT Agent → Orion Context Broker
```

## Error Responses

The API returns an error in the following situations:

* Invalid or missing JSON payload.
* Missing `apikey`.
* Missing `deviceid`.
* No telemetry attributes provided.
* Unsupported `agent` value.

## Example Responses

### Successful request

```json
{
  "status": "ok",
  "mode": "json-http",
  "fiware_status": 204
}
```

### Missing API Key

```json
{
  "error": "apikey e deviceid obrigatórios"
}
```

### Missing telemetry data

```json
{
  "error": "sem dados de sensores"
}
```

### Invalid agent type

```json
{
  "error": "agent inválido"
}
```
